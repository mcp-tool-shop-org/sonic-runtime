# sonic-runtime Wire Protocol

## Transport

Newline-delimited JSON over stdio (same shape as MCP stdio transport).

- **stdin** — sonic-core sends requests to the runtime
- **stdout** — runtime sends responses back (protocol messages only)
- **stderr** — runtime diagnostic logging (never protocol)

One JSON object per line. No pretty-printing. No trailing commas.

## Request Envelope

```json
{ "id": 1, "method": "play", "params": { "handle": "h_000000000001", "volume": 0.8 } }
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | int | yes | Request ID for correlation |
| `method` | string | yes | Method name |
| `params` | object | no | Method-specific parameters |

## Response Envelope (success)

```json
{ "id": 1, "result": { "handle": "h_000000000001", "duration_ms": 180000 } }
```

## Response Envelope (error)

```json
{ "id": 1, "error": { "code": "device_unavailable", "message": "Requested output device is no longer available.", "retryable": true } }
```

Error shape matches `SonicError` from `@sonic-core/types`.

## Methods

### Lifecycle

| Method | Params | Result | Description |
|--------|--------|--------|-------------|
| `version` | — | `{ name, version, protocol }` | Handshake / health check |
| `shutdown` | — | — | Graceful exit |

### Playback

| Method | Params | Result | Description |
|--------|--------|--------|-------------|
| `load_asset` | `{ asset_ref }` | `{ handle, duration_ms? }` | Load audio file, return opaque handle |
| `play` | `{ handle, volume?, pan?, fade_in_ms?, loop? }` | `null` | Start playback |
| `pause` | `{ handle, fade_out_ms? }` | `null` | Pause |
| `resume` | `{ handle, fade_in_ms? }` | `null` | Resume |
| `stop` | `{ handle, fade_out_ms? }` | `null` | Stop and release |
| `seek` | `{ handle, position_ms }` | `null` | Seek to position |
| `set_volume` | `{ handle, level, fade_ms? }` | `null` | Set volume with optional ramp |
| `set_pan` | `{ handle, value, ramp_ms? }` | `null` | Set pan with optional ramp |
| `get_position` | `{ handle }` | `{ position_ms }` | Current position |
| `get_duration` | `{ handle }` | `{ duration_ms }` | Total duration (null for streams) |

### Devices

| Method | Params | Result | Description |
|--------|--------|--------|-------------|
| `list_devices` | — | `DeviceInfo[]` | Enumerate output devices |
| `set_device` | `{ device_id }` | `null` | Switch output device |

### Synthesis

| Method | Params | Result | Description |
|--------|--------|--------|-------------|
| `synthesize` | `{ engine, voice, text, speed? }` | `{ handle }` | Run TTS, return playable handle |

## Handle Semantics

Handles are opaque strings generated by the runtime (format: `h_` + hex).
They are **never exposed outside sonic-runtime**. sonic-core's `SidecarBackend`
translates `handle` ↔ `playback_id`. Clients never see handles.

## Error Codes

| Code | Description |
|------|-------------|
| `invalid_params` | Missing or malformed parameters |
| `method_not_found` | Unknown method name |
| `playback_not_found` | Handle does not exist |
| `device_unavailable` | Requested device not found or unplugged |
| `seek_unsupported` | Cannot seek this source type |
| `synthesis_validation_failed` | Bad engine/voice/text |
| `internal_error` | Unexpected runtime error |

## Logging Policy

All diagnostic output goes to **stderr**, never stdout.
Format: `[sonic-runtime] message`

The runtime must never write anything to stdout that is not a valid JSON response.
A stray `Console.WriteLine` on stdout corrupts the protocol stream.
